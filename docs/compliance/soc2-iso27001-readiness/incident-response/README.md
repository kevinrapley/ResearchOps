# ResearchOps incident response readiness

This folder holds the incident response evidence pack for the ResearchOps platform.

It supports SOC 2 and ISO/IEC 27001 readiness. It does not assert SOC 2 compliance or ISO/IEC 27001 certification.

## Artefacts

| Artefact | Purpose | Readiness status |
| --- | --- | --- |
| `incident-response-runbooks.md` | Defines service-specific runbooks for likely ResearchOps security, privacy and operational incidents. | Drafted for security, privacy and service-owner review. |
| `personal-data-breach-handling.md` | Defines the decision route for suspected personal data breaches, including escalation, notification assessment and evidence to retain. | Drafted for Home Office privacy and security review. |
| `incident-exercise-record.md` | Defines the tabletop or simulated incident test that must be completed before the process can be treated as exercised evidence. | Planned, not yet completed. |

## Evidence position

The runbooks and breach handling process are now partially evidenced because they exist in version-controlled service documentation.

The control is not complete until:

- Home Office service owner, information asset owner, senior risk owner, privacy and security roles have reviewed and approved the process
- an incident response exercise has been run
- exercise outcomes, timings, decisions, lessons and follow-up actions have been recorded
- the compliance evidence index links to the completed exercise record

Until then, ResearchOps can describe this as incident response readiness evidence. It must not be described as an exercised or independently assured incident response control.
