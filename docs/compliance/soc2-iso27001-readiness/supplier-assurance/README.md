# ResearchOps supplier assurance readiness

This folder records the supplier and integration assurance evidence needed before ResearchOps can support a SOC 2 or ISO/IEC 27001 claim.

It does not assert supplier approval, contract approval, DPIA completion or independent assurance. It records the supplier dependencies that need Home Office service-owner, commercial, privacy and security review.

## Artefacts

| Artefact | Purpose | Status |
| --- | --- | --- |
| `supplier-assurance-register.md` | Defines the supplier and integration assurance register for Cloudflare, GitHub, Airtable, Mural, email delivery and configured AI services. | Drafted for review. |

## Evidence position

The supplier assurance control is now partially evidenced because the supplier dependencies, data flows and review questions are named in version-controlled service documentation.

The control is not complete until:

- each supplier has an agreed ResearchOps role and data-processing position
- contract, DPIA and data-sharing positions have been confirmed
- locations, subprocessor dependencies and transfer positions have been recorded
- security assurance evidence has been reviewed and dated
- service owner, information asset owner, privacy, security and commercial roles have signed off the register
- the register has an owner and review cadence

Until then, ResearchOps can describe this as supplier assurance readiness evidence. It must not be described as approved supplier assurance or completed third-party risk management.
