# Pace Layers Matrix

Use this matrix when the sourcebook feels too large to adopt in one move. The aim is to separate durable operating decisions from faster experiments, then turn the learning from fast work into better rules, guidance and investment choices.

## How to use it

1. Name the research operations pressure point.
2. Map it to one or more of the 8 pillars.
3. Decide which pace layer it belongs to.
4. Choose the sourcebook clauses that govern the next decision.
5. Record the adoption action, owner, review date and evidence.

## Matrix headings

| Pace layer | Pillars | Adoption question | Sourcebook use | Next action | Evidence |
| --- | --- | --- | --- | --- | --- |
| Purpose and strategic intent | ORG-CONT, GOVERN | What must research operations make possible? | Set sponsorship, authority, funding and ethical boundaries. | | |
| Rules, decision rights and scope | GOVERN, SCOPE | Which decisions need clearer rules? | Set intake, triage, approval, exception and escalation routes. | | |
| Infrastructure and knowledge systems | INFRA-PROV, DATA-STO-ACC | Which shared systems need to become stable? | Prioritise tooling, repository, access, retention, taxonomy and integration work. | | |
| Participant operations and research settings | REC-ADMN, ENVIRO | Where are participants or researchers feeling avoidable friction or risk? | Improve sourcing, consent, scheduling, participant care, accessibility, fieldwork and remote settings. | | |
| Practice, support and habits | PEOP-COMM, SCOPE | What can teams practise or standardise now? | Shape coaching, community standards, self-service support, methods guidance and operating learning. | | |

## Review rhythm

Review the matrix quarterly. Fast work should produce evidence that updates slower rules, standards and investment decisions. Slow work should give fast teams the constraints they need to move with confidence.
