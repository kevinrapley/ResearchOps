# Design critiques

Design critique records must be stored in dated folders using this path pattern:

```text
docs/design-critiques/YY/MM/DD/
```

Use the date the critique was created or approved for repository storage. For example, a critique recorded on 7 May 2026 belongs in:

```text
docs/design-critiques/26/05/07/
```

## Rules

- Store new design critique records inside the dated folder pattern.
- Use `YY`, `MM` and `DD` as two-digit values.
- Keep filenames descriptive and stable.
- Add or update a dated folder `README.md` when adding records.
- Keep root-level design critique documents out of this directory except this `README.md`.
