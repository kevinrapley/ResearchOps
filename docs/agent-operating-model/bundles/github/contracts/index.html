Agent evidence schema model

This representative shape shows how the bundle makes evidence checkable.

{
  "$schema": "https://json-schema.org/draft/2020-12/schema",
  "title": "Agent evidence",
  "type": "object",
  "required": [
    "task",
    "repository",
    "mode",
    "commands",
    "changed_files",
    "validation"
  ],
  "properties": {
    "task": { "type": "string" },
    "repository": { "type": "string" },
    "mode": { "type": "string" },
    "commands": {
      "type": "array",
      "items": {
        "type": "object",
        "required": ["command", "status"],
        "properties": {
          "command": { "type": "string" },
          "status": { "enum": ["pass", "fail", "not-run"] },
          "reason": { "type": "string" }
        }
      }
    },
    "changed_files": {
      "type": "array",
      "items": { "type": "string" }
    }
  }
}

How the agent uses this

The agent records commands and outcomes. A validation claim without evidence should fail grading.

Downstream use

Graders and scripts use schema-valid evidence as the input to release readiness, PR readiness and conformance checks.
