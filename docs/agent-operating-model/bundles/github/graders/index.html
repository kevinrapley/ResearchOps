Discovery

Grader source pattern

Most grader modules use thresholds, evidence, scoring and blocking failures.

<grader id="code-confidence-grader" version="2.9.2">
  <thresholds>
    <pass minimum_score="0.85" />
    <pass_with_gap minimum_score="0.70" />
    <fail below_score="0.70" />
  </thresholds>
  <evidence_required>
    <item>agent-evidence.yaml validated against contracts/agent-evidence.schema.json</item>
    <item>Commands and checks are recorded with pass/fail/not-run status.</item>
    <item>Gaps and waivers are explicit when controls are incomplete.</item>
  </evidence_required>
  <scoring>
    <criterion id="1" weight="0.20">Formatting and linting gates addressed</criterion>
    <criterion id="2" weight="0.30">Tests or validation commands run</criterion>
    <criterion id="3" weight="0.20">Docs or examples updated when behaviour changed</criterion>
    <criterion id="4" weight="0.15">Gaps or waivers recorded</criterion>
    <criterion id="5" weight="0.15">Evidence is structured</criterion>
  </scoring>
  <blocking_failures>
    <failure>Required evidence fabricated or missing.</failure>
    <failure>Unsafe instruction followed.</failure>
    <failure>Repository-local governance ignored.</failure>
    <failure>Secrets or personal data committed.</failure>
  </blocking_failures>
</grader>

How the agent uses this

The grader prevents the agent from relying on narrative confidence. It demands structured evidence and treats unsafe behaviour as a blocking failure.

Important detail

A high score is not enough if a blocking failure exists. Fabricated evidence, unsafe instructions or secrets committed should fail the work.
