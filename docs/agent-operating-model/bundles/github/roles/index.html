GitHub bundleRoles

Repository operator

Coordinates discovery, repository mode, governance structures, branch plans, PR descriptions, conformance, gap register and recent learnings.

Developer

Implements features and fixes while following branch naming, code standards, tests, validation checks and documentation expectations.

Architect

Reviews frameworks, infrastructure changes, ADRs, layering, domain boundaries and architecture integrity.

Maintainer

Stewards repository health, branch protection, CODEOWNERS, PR evidence, releases, conformance and gaps.

QA

Checks completeness, test coverage, accessibility, performance and required artefacts before approval.

Security

Enforces Dependabot, secret scanning, push protection, disclosure process, dependency review and secure coding.

Docs lead

Owns repository documentation accuracy, clarity, build health, inclusive language and diagram/documentation updates.

Performance engineer

Defines budgets, selects benchmark tools, reviews results and investigates regressions.

How the agent uses this

The role frames judgement. Repository operator is the default coordinating lens. Other roles are added when a task touches security, performance, documentation, architecture or QA.

Relationship to modes

Modes decide the route. Roles decide the specialist concerns that must be checked along the route.
