# Release-ready seed fixture

This repository is a seed fixture for release-readiness evals.

It represents a small repository that already has some release evidence, but still requires the agent to inspect whether the release is actually ready.

## Fixture role

This is input material, not a completed release output.

The fixture gives the agent enough evidence to practise release-readiness reasoning: it has a changelog and a CI workflow, but it should still require explicit checks for rollback, provenance, SBOM evidence and unresolved blockers.

## Expected agent interpretation

The agent should detect:

- a repository with a release history signal from `CHANGELOG.md`
- a CI workflow at `.github/workflows/ci.yml`
- no completed release checklist unless one is generated by the eval
- no rollback plan unless one is generated by the eval
- no provenance record unless one is generated by the eval
- no SBOM artefact unless one is generated by the eval

The correct mode for a release-readiness eval is `repo-release`.

## Expected follow-on output

A completed release-readiness output should include:

- updated changelog evidence
- release checklist
- rollback plan
- metadata provenance
- SBOM artefact
- release decision
- any blocking gaps

Those completed artefacts belong in the generated eval output, not in this seed fixture.

## Boundary

Do not treat this fixture as production-ready. It is deliberately incomplete enough to force the agent to make a release judgement based on evidence rather than file presence alone.
