# Security Policy

## Supported Versions

We release new versions regularly and support the most recent major version.  Older versions may receive critical security fixes at the maintainers’ discretion.

## Reporting a Vulnerability

If you believe you have found a security vulnerability, please **do not** open a public issue.  Instead, follow these steps:

1. Check whether this repository has **private vulnerability reporting** enabled.  If so, use the “Report a vulnerability” button on the repository’s Security tab.
2. If private reporting is not available, email the maintainers at the address listed below with a detailed description of the vulnerability, steps to reproduce and any potential mitigation.

We will acknowledge your report within three working days and keep you informed of the resolution progress.  We appreciate responsible disclosure and will credit you in release notes if desired.

## Dependent Projects

If this project is a dependency of your software and we have published a fix for a vulnerability, please upgrade to the patched version as soon as possible.  Dependabot alerts will also notify you of vulnerable dependencies.
